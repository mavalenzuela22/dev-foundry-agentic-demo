# Experiment Model

This repository has two related but different concerns.

## Primary Subject

The primary subject is Dev Foundry powered by Alita / EliteA agents.

The main experiment is to validate whether a small agent system can coordinate software engineering work in a governed way.

## Test Application

Foundry Request Board is the test application.

It is the target repo app used as the guinea pig for the agent experiments.

The app exists so the Alita / EliteA agents have something concrete to inspect, modify, validate, and report on.

## Why This Matters

Without a small app, the demo would only talk about agents.

With a small app, the demo can show agents doing real work against code, tests, and documentation.

## Boundary

Foundry Request Board is intentionally small.

It should help the agent demo, not become the main complexity of the demo.
