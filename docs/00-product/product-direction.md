# Product Direction

Product name: Dev Foundry Agentic Demo Kit

Demo app: Foundry Request Board

Audience: software engineers with early AI tooling experience, around three months of usage.

Purpose: demonstrate a small governed agent workflow for software engineering.

The demo must be simple enough to explain quickly and strong enough to show real code changes, validation, and reporting.

Key idea: Dev Foundry is not only a chat assistant. It is a governed engineering workflow where specialized agents collaborate under clear rules.

The demo app will classify software change requests by type, risk, recommended mode, and reason.

The first planned feature is support for classifying documentation-only requests as low risk and ready for bounded execution.

The demo should avoid heavy frameworks, backend services, databases, authentication, deployment, or secrets.
